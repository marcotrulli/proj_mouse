🛠 Script Python per leggere l’MPU6050 su Raspberry Pi (I2C)

Usa la libreria Python “mpu6050-raspberrypi” o simile, che semplifica la lettura.

1) Installa i prerequisiti

Sul tuo Raspberry Pi, abilita I2C:

sudo raspi-config
# -> Interfacing Options -> I2C -> Yes
sudo apt update
sudo apt install python3-pip python3-smbus i2c-tools
pip3 install mpu6050-raspberrypi

Verifica che l’IMU sia collegata e risponda su I2C (indirizzo tipico 0x68):

i2cdetect -y 1
2) mpu_mouse.py – Script per leggere dati
#!/usr/bin/env python3
# mpu_mouse.py
# Lettura dati MPU6050 su Raspberry Pi

from mpu6050 import mpu6050
import time

# Indirizzo I2C (tipicamente 0x68)
mpu = mpu6050(0x68)

print("Inizio lettura dati...")

try:
    while True:
        accel = mpu.get_accel_data()
        gyro  = mpu.get_gyro_data()

        # Accelerazione in g ~ X, Y, Z
        print(f"Accel | X:{accel['x']:.2f}g  Y:{accel['y']:.2f}g  Z:{accel['z']:.2f}g")
        # Giroscopio in °/s
        print(f"Gyro  | X:{gyro['x']:.2f}  Y:{gyro['y']:.2f}  Z:{gyro['z']:.2f}")

        print("-" * 50)
        time.sleep(0.2)

except KeyboardInterrupt:
    print("\nFine programma.")

#!/usr/bin/env python3
# mpu_mouse.py
# Lettura dati MPU6050 su Raspberry Pi (I2C)

from mpu6050 import mpu6050
import time

# Inizializza sensore (indirizzo I2C tipico 0x68)
sensor = mpu6050(0x68)

print("Inizio lettura dati MPU6050... Premi Ctrl+C per fermare.")

try:
    while True:
        accel = sensor.get_accel_data()
        gyro  = sensor.get_gyro_data()

        # Accelerazione in g
        print(f"Accel | X:{accel['x']:.2f}g  Y:{accel['y']:.2f}g  Z:{accel['z']:.2f}g")
        # Giroscopio in °/s
        print(f"Gyro  | X:{gyro['x']:.2f}  Y:{gyro['y']:.2f}  Z:{gyro['z']:.2f}")

        print("-" * 50)
        time.sleep(0.2)

except KeyboardInterrupt:
    print("\nProgramma terminato.")

Il problema è che i due pin I2C standard del Raspberry Pi (SDA = GPIO2, SCL = GPIO3) sono già usati da un altro dispositivo. I2C, in generale, è un bus condiviso, ma tutti i dispositivi sullo stesso bus devono avere indirizzi diversi.

Ecco le opzioni che ti ho citato, spiegate in parole semplici:

1️⃣ Condividere lo stesso bus I2C

Immagina i pin SDA/SCL come un cavo condiviso.

Puoi collegare più dispositivi sullo stesso cavo, ma ognuno deve avere un indirizzo diverso.

Se l’altro dispositivo ha già indirizzo 0x68 (come l’MPU6050), non puoi usarlo insieme senza cambiare l’indirizzo.

2️⃣ Usare un secondo bus I2C

Alcuni Raspberry Pi hanno un secondo bus I2C su altri pin (es. GPIO0/GPIO1), oppure si può creare via software usando librerie come pigpio.

Così l’MPU6050 avrà un cavo separato, senza interferire con l’altro dispositivo.

3️⃣ Usare un multiplexer I2C (tipo TCA9548A)

È un piccolo chip che funziona come uno switch per l’I2C.

Permette di avere più sensori sullo stesso paio di pin senza conflitti di indirizzo, perché crea 8 “canali separati” sullo stesso bus.

In pratica, puoi collegare l’altro dispositivo e l’MPU6050 allo stesso SDA/SCL e scegliere quale canale leggere dal Raspberry Pi.

GND	Massa	GND (Pin 6)	Fondamentale, collega il sensore a massa!
SDA	Dati I2C	GPIO2 (Pin 3)	Condiviso con l’altro dispositivo
SCL	Clock I2C	GPIO3 (Pin 5)	Condiviso con l’altro dispositivo
AD0	Indirizzo I2C	GND	Indirizzo 0x68
INT	Interrupt	lasciato scollegato	opzionale
Note importanti per la condivisione:

Gli indirizzi devono essere diversi: se l’altro dispositivo occupa 0x68, cambia l’AD0 dell’MPU6050 a VCC per usare 0x69.

Collegare GND è obbligatorio: tutti i dispositivi sul bus devono avere massa comune, altrimenti I2C non funziona.

Il codice Python non cambia: basta che l’indirizzo nel costruttore corrisponda a quello impostato su AD0.

Poi controlla:

sudo systemctl status bluetooth

3️⃣ Prova di nuovo a lanciare bluetoothctl

bluetoothctl

Se funziona, vedrai il prompt [bluetooth]# invece di Waiting to connect….

4️⃣ Se ancora non funziona, spesso basta spegnere e riaccendere l’interfaccia Bluetooth:

Salvalo ad esempio come auto_bluetooth.sh e rendilo eseguibile (chmod +x auto_bluetooth.sh).

#!/bin/bash
# auto_bluetooth.sh
# Script per collegare automaticamente il Raspberry Pi al PC THE-MARCO-PC
# senza richiesta di conferma passkey

# MAC address del PC
PC_MAC="78:46:5C:11:AA:48"

echo "Avvio servizio Bluetooth..."
sudo systemctl start bluetooth

echo "Apertura bluetoothctl..."
bluetoothctl << EOF
power on
discoverable on
pairable on

# Imposta agente senza input/output (non chiede conferma passkey)
agent NoInputNoOutput
default-agent

# Prova a fare trust del PC
trust $PC_MAC

# Prova a connettere il PC
connect $PC_MAC
EOF

echo "Script completato. Se il PC è acceso e vicino, dovrebbe essere connesso senza passkey."

2️⃣ Abilitare HID (Mouse/Keyboard)

Abilita bluetoothd con supporto HID:

sudo nano /etc/systemd/system/bluetooth.target.wants/bluetooth.service

Trova la riga:

ExecStart=/usr/lib/bluetooth/bluetoothd

E modificala in:

ExecStart=/usr/lib/bluetooth/bluetoothd --noplugin=sap --experimental

L’opzione --experimental permette di usare HID Profile.

Poi salva (Ctrl+O) e riavvia:

sudo systemctl daemon-reexec
sudo systemctl restart bluetooth
3️⃣ Creare un device HID virtuale

Puoi usare python3 e la libreria pydbus per emulare un mouse:

Esempio minimale:

# virtual_mouse.py
from pydbus import SystemBus
import time

bus = SystemBus()
manager = bus.get("org.bluez", "/org/bluez")

adapter = manager.Adapter1
device_path = "/org/bluez/hci0/dev_78_46_5C_11_AA_48"

device = bus.get("org.bluez", device_path)

# Connetti al PC
device.Connect()

# Emula un click sinistro
device.SendReport([0x00, 0x01, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00], 0x01)
time.sleep(1)

# Disconnetti
device.Disconnect()

mouse_clien.py PRESENTE NELLA REPOSTORY:
#!/usr/bin/python
#
# Reads local mouse events and forwards them to the btk_server DBUS service
#
#

import dbus
import dbus.service
import dbus.mainloop.glib
import time
import evdev  # used to get input from the mouse
from evdev import InputDevice, ecodes


# define a client to listen to local mouse events
class Mouse:
    def __init__(self):
        # the structure for a bluetooth mouse input report (size is 6 bytes)

        self.state = [
            0xA1,  # this is an input report
            0x02,  # Usage report = Mouse
            0x00,  # Bit array for Buttons ( Bits 0...4 : Buttons 1...5, Bits 5...7 : Unused )
            0x00,  # Rel X
            0x00,  # Rel Y
            0x00,  # Mouse Wheel
        ]

        print("Setting up DBus Client")

        self.bus = dbus.SystemBus()
        self.bluetoothservice = self.bus.get_object('org.upwork.HidBluetoothService', "/org/upwork/HidBluetoothService")
        self.iface = dbus.Interface(self.bluetoothservice, 'org.upwork.HidBluetoothService')

        print("Waiting for mouse")

        # keep trying to find a mouse
        have_dev = False
        count = 0
        NUMBER_OF_TRIES = 100

        while have_dev is False and count < NUMBER_OF_TRIES:
            try:
                # try and get a mouse - loop through all devices and try to find a mouse
                devices = [evdev.InputDevice(fn) for fn in evdev.list_devices()]
                for device in reversed(devices):
                    if "mouse" in device.name.lower():
                        print("Found a keyboard with the keyword 'mouse'")
                        print("device name is " + device.name)
                        self.dev = InputDevice(device.fn)
                        have_dev = True
                        break
            except OSError:
                print("Mouse not found, waiting 3 seconds and retrying")
                time.sleep(3)
            count += 1
        
        if not have_dev:
            print("Mouse not found after " + str(NUMBER_OF_TRIES) + " tries.")
            return
        else:
            print("Mouse Found")
            print("Starting mouse event loop")
            self.event_loop()

    # take care of mouse buttons
    def change_state_button(self, event):
        if event.code == ecodes.BTN_LEFT:
            print("Left Mouse Button Pressed")
            self.state[2] = event.value
        elif event.code == ecodes.BTN_RIGHT:
            print("Right Mouse Button Pressed")
            self.state[2] = 2 * event.value
        elif event.code == ecodes.BTN_MIDDLE:
            print("Middle Mouse Button Pressed")
            self.state[2] = 3 * event.value
        self.state[3] = 0x00
        self.state[4] = 0x00
        self.state[5] = 0x00

    # take care of mouse movements
    def change_state_movement(self, event):
        if event.code == ecodes.REL_X:
            self.state[3] = event.value & 0xFF
        elif event.code == ecodes.REL_Y:
            self.state[4] = event.value & 0xFF
        elif event.code == ecodes.REL_WHEEL:
            self.state[5] = event.value & 0xFF

    # poll for mouse events
    def event_loop(self):
        for event in self.dev.read_loop():
            if event.type == ecodes.EV_KEY and event.value < 2:
                self.change_state_button(event)
            elif event.type == ecodes.EV_REL:
                self.change_state_movement(event)
            try:
                self.send_input()
            except:
                print("Couldn't send mouse input")
                break

    # forward mouse events to the dbus service
    def send_input(self):
        self.iface.send_mouse(self.state[2], self.state[3:6])



if __name__ == "__main__":
    print("Setting up mouse")
    Mouse()

Perfetto, grazie per il log dettagliato.

Il problema adesso non è più Python o le librerie, ma il fatto che il client non trova il servizio DBus org.upwork.HidBluetoothService.
Ecco cosa significa e come procedere:

🔹 Cosa sta succedendo

Nel tuo mouse_client.py hai questa riga:

self.bluetoothservice = self.bus.get_object('org.upwork.HidBluetoothService', "/org/upwork/HidBluetoothService")

org.upwork.HidBluetoothService è il nome del servizio DBus a cui il client cerca di connettersi.

L’errore:

org.freedesktop.DBus.Error.ServiceUnknown: The name org.upwork.HidBluetoothService was not provided by any .service files

significa nessun processo attivo ha registrato quel servizio DBus.
In altre parole: il tuo “server” HID Bluetooth (btk_server) non è in esecuzione, oppure non è installato/registrato come servizio.

🔹 Come risolvere

Assicurati che il servizio HID Bluetooth sia attivo

Se il tuo progetto usa un servizio DBus custom, deve esserci un .service file in /usr/share/dbus-1/system-services/ o /etc/dbus-1/system.d/ che definisca:

[D-BUS Service]
Name=org.upwork.HidBluetoothService
Exec=/usr/bin/python3 /percorso/al/tuo/btk_server.py
User=root

Poi puoi avviare il server con:

sudo systemctl start btk_server.service

Oppure direttamente dal terminale:

sudo python3 btk_server.py

Verifica che il servizio sia registrato su DBus

dbus-send --system --dest=org.upwork.HidBluetoothService --type=method_call --print-reply /org/upwork/HidBluetoothService org.freedesktop.DBus.Introspectable.Introspect

Se risponde, il servizio è attivo e pronto per ricevere input dal client.

Assicurati che il servizio parta prima del client

Il client mouse_client.py deve partire dopo che il server HID è attivo, altrimenti DBus non troverà nulla.

🔹 In sintesi

Il tuo Python client funziona correttamente.

Devi avviare prima il server HID (btk_server o HidBluetoothService) che registri il nome org.upwork.HidBluetoothService su DBus.

Solo dopo puoi far partire mouse_client.py.

Se vuoi, posso scriverti un piccolo script di test che fa partire automaticamente il server e poi il client, così eviti l’errore DBus.
